Complexity Report
  Info ComplexityReport.js

A tool for reporting code complexity metrics in JavaScript projects. Go to http://jscomplexity.org/complexity for extendend information about what complexity means in this context. Currently the tool reports on:

  • lines of code;
  • cyclomatic complexity;
  • Halstead metrics;
  • maintainability index.

The tool, when deployed as an automated continuous integration system, can be configured to fail when complexity metrics pass a specified threshold. You can head off build breakage by using this extension.

 Options ComplexityReport.js

You can adjust the settings by modifying the ./config.js file located in the base folder of this extension. The following options are available:

  • logicalor: Boolean indicating whether operator || should be considered a source of cyclomatic complexity, defaults to true.
  • switchcase: Boolean indicating whether switch statements should be considered a source of cyclomatic complexity, defaults to true.
  • forin: Boolean indicating whether for...in loops should be considered a source of cyclomatic complexity, defaults to false.
  • trycatch: Boolean indicating whether catch clauses should be considered a source of cyclomatic complexity, defaults to false.
  • newmi: Boolean indicating whether the maintainability index should be rebased on a scale from 0 to 100.

The report contains properties detailing the complexity of each function from the source code. There is also a maintainability index as well as aggregate complexity metrics for the source in its entirety. To learn more about the JSComplexity.org and complexityReport.js select CRJS Info
